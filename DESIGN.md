# 沼气提纯站项目全生命周期管理系统 - 设计文档

## 项目概述

本系统是一个专注于沼气提纯站项目的全生命周期管理工具，提供从项目跟踪、签约、手续办理、建设到运营的全流程管理。支持项目人员配置评估、地图可视化、甘特图展示等功能。

## 更新日志

### v1.1.1 - 关键Bug修复：数据保存问题 (2026-03-28)

#### 严重问题
项目管理页的所有修改和新建操作都无法保存，导致地图视图和甘特图空白。

#### 修复内容

##### 1. 数据存储层修复 (`js/dataStore.js`)
- **问题**: 数据保存后无法正确读取，LocalStorage 操作缺乏错误处理
- **修复**:
  - 新增 `DATA_VERSION` 数据版本控制，支持数据迁移
  - 重写 `saveProject()` 方法，确保正确处理新建和更新操作
  - 新增 `deepMerge()` 方法，避免更新时丢失原有数据字段
  - 增强 `saveToStorage()` 方法，添加详细的错误处理和日志
  - 新增 `init()` 方法自动初始化存储键，避免 null 错误
  - 所有数据操作添加详细的控制台日志，方便调试

##### 2. 表单提交逻辑修复 (`js/app.js`)
- **问题**: `performSave()` 方法总是调用 `createProject`，编辑时也应该调用 `updateProject`
- **修复**:
  ```javascript
  // 根据是否有ID判断是新建还是更新
  if (projectData.id) {
      result = projectManager.updateProject(projectData.id, projectData);
  } else {
      result = projectManager.createProject(projectData);
  }
  ```
- **人员保存同样修复**: `savePersonnel()` 方法同样区分新建和更新操作

##### 3. 数据流向验证
- 表单输入 → 数据收集 → 存储到 LocalStorage → 读取显示
- 整个数据流向已验证无断点

#### 新增功能

##### 5. 人员负荷评估页面
创建了新页面`personnel-assessment.html`，包含以下功能：

- **项目量合理性评估**: 
  - 统计每个人在指定时间段内负责的项目数量
  - >5个项目标记为红色警告（过载）
  - >3个项目显示黄色提醒

- **项目间距离分析**: 
  - 计算每个人负责的项目之间的实际距离
  - >300km标记为黄色提醒
  - >500km标记为红色警告
  - 使用Haversine公式计算两点间距离

- **技能匹配度分析**: 
  - 对比人员熟悉技能与项目所需工艺
  - 计算匹配度百分比（0-100%）
  - 低于70%显示警告建议

- **综合评分**: 
  - 给出每个人员配置的综合健康度评分（0-100%）
  - 90-100%: 优秀（绿色）
  - 80-89%: 良好（蓝色）
  - 60-79%: 一般（黄色）
  - <60%: 需优化（红色）

- **时间维度**: 
  - 支持选择评估时间点或时间段
  - 默认评估未来9个月的时间段

- **可视化图表**:
  - 人员项目负荷分布柱状图
  - 健康度评分分布饼图
  - 技能匹配度低的人员详情列表

## 功能模块

### 1. 项目管理

#### 核心功能
- 项目创建、编辑、删除
- 项目列表展示与筛选
- 项目详情查看
- 项目生命周期管理（5个阶段）

#### 项目生命周期阶段
| 阶段 | 默认时长 | 颜色 | 说明 |
|------|---------|------|------|
| 跟踪 | 30天 | 橙色 | 项目前期跟踪 |
| 签约 | 30天 | 蓝色 | 合同签订 |
| 手续 | 120天 | 紫色 | 政府手续办理 |
| 建设 | 150天 | 粉色 | 工程建设 |
| 运营 | 365天 | 绿色 | 项目运营 |

#### 项目属性
- **基本信息**: 名称、位置（省市区+详细地址）
- **技术参数**: 设计规模、日产量、电/蒸汽价格
- **处理工艺**: 多选（脱硫工艺 + 脱碳工艺）
- **其他信息**: 备注/notes字段
- **生命周期**: 各阶段的起止日期和牵头人

### 2. 人员管理

#### 核心功能
- 人员创建、编辑、删除
- 人员列表展示（按部门筛选）
- 人员详情查看
- 工作负荷计算

#### 人员属性
- **基本信息**: 姓名、性别、年龄、部门、职位

- **联系**: 电话、邮箱
- **配置**: 最大可承担项目数

#### 部门列表

- 财务部、行政部、采购部、质量部

### 4. 甘特图

- 按项目视图：展示各项目阶段时间线

### 5. 人员负荷评估（新增）

#### 评估指标
| 指标 | 阈值 | 状态 |
|------|------|------|
| 项目数量 | >5 | 红色警告 |
| 项目数量 | 4-5 | 黄色提醒 |
| 项目间距离 | >500km | 红色警告 |
| 项目间距离 | 300-500km | 黄色提醒 |
| 技能匹配度 | <50% | 红色警告 |
| 技能匹配度 | 50-70% | 黄色提醒 |

#### 健康度评分算法
```
基础分 = 100
- 项目数 > 5: 每超1个扣15分
- 项目数 > 上限: 每超1个扣10分
+ 技能匹配度加分: (匹配度 - 70) * 0.2
- 距离 > 500km: 扣20分
- 距离 300-500km: 扣10分
最终得分 = clamp(0, 100, 计算结果)
```

### 6. 智能推荐

#### 功能
- 按距离聚合项目群（150km）
- 推荐阶段牵头人
- 检查人员配置合理性

#### 推荐算法权重
- 工艺匹配: 25%
- 经验值: 20%
- 距离因子: 20%
- 负载因子: 20%
- 能力评分: 15%
